# HDGP 技术债与安全加固清单

本文档列出 HDGP 参考实现当前已知的**技术债**与**安全加固项**，供 Alpha / Beta 阶段逐项推进与审计使用。  
完成状态可勾选更新（☐ 未做 / ☑ 已做 / ➖ 不适用或暂缓）。

---

## 一、技术债（Technical Debt）

### 1.1 规则与行为覆盖

| 编号 | 描述 | 优先级 | 状态 | 备注 |
|------|------|--------|------|------|
| TD-R1 | 扩展规则集：多语言/多地区敏感词与表述模式 | 中 | ☐ | 当前以中英关键词为主，可增加 locale 与多语检测 |
| TD-R2 | 规则可配置化：将部分阈值/关键词从代码抽到可加载配置（仍只读、签名） | 中 | ☐ | 不改变“运行时不可注入”的约束 |
| TD-R3 | 不确定性熔断：引入置信度/多次采样差异度评分，低于阈值时熔断 | 中 | ☐ | 阶段 1 规划中有，尚未实现 |
| TD-R4 | 规则冲突时的完整优先级表与可配置覆盖策略 | 低 | ☐ | 当前为固定顺序，见 `HDGP_CORE_MAPPING_SPEC.md` §3.3 |

### 1.2 审计与可观测性

| 编号 | 描述 | 优先级 | 状态 | 备注 |
|------|------|--------|------|------|
| TD-A1 | 审计日志持久化：当前为内存环形 buffer，需支持落盘或只追加日志 | 高 | ☐ | Alpha 建议实现，便于合规与复盘 |
| TD-A2 | 审计查询接口：按 request_id、时间范围、verdict、rule_id 等过滤 | 中 | ☐ | 当前仅支持 `limit` 取最近 N 条 |
| TD-A3 | 暴露关键指标（请求量、熔断率、规则命中统计）供 Prometheus/观测栈 | 低 | ☐ | Beta 阶段可做 |

### 1.3 网关与集成

| 编号 | 描述 | 优先级 | 状态 | 备注 |
|------|------|--------|------|------|
| TD-G1 | 网关层与 Engine 层解耦：将“候选输出构造/占位”从 `internal/gateway/chat.go` 抽成清晰适配层 | 中 | ☐ | 便于后续多入口、多 Skill 接入 |
| TD-G2 | 降级策略：HDGP 不可用时的 fail-closed / 超时 / 健康检查 | 中 | ☐ | 见 `spec/HDGP_INTEGRATION_SPEC.md` §6 |
| TD-G3 | 多模型/多后端适配器与统一 Skill 封装 | 低 | ☐ | Beta 多场景支持 |

### 1.4 配置与部署

| 编号 | 描述 | 优先级 | 状态 | 备注 |
|------|------|--------|------|------|
| TD-C1 | 监听地址/端口、审计 buffer 大小等可配置（环境变量或配置文件） | 低 | ☐ | 当前端口写死 `:8080` |
| TD-C2 | 正式构建与发布：二进制构建脚本、版本号注入、容器镜像 | 低 | ☐ | 便于生产部署 |

## 二、安全加固（Security Hardening）

### 2.1 访问控制与输入安全

| 编号 | 描述 | 优先级 | 状态 | 备注 |
|------|------|--------|------|------|
| SH-A1 | API 认证与鉴权：对 `/evaluate`、`/audit`、`/appeal` 等接口做调用方身份与权限控制 | 高 | ☐ | 对外开放前必须，可放 Gateway 或反向代理 |
| SH-A2 | 速率限制与配额：防止滥用与 DoS，可按 IP/AppID/租户限流 | 高 | ☐ | 建议在 Gateway 或上游实现 |
| SH-A3 | 请求体大小与深度限制：防止超大 JSON 或深层嵌套导致资源耗尽 | 中 | ☐ | 当前未做显式限制 |
| SH-A4 | 敏感字段脱敏：审计日志中对 PII、token 等做脱敏或哈希 | 中 | ☐ | 视合规要求而定 |

### 2.2 规则包与完整性

| 编号 | 描述 | 优先级 | 状态 | 备注 |
|------|------|--------|------|------|
| SH-P1 | 规则包签名校验：Engine 加载规则包时强制验证签名（伦理基线 §7.2） | 高 | ☐ | 当前为占位字段，无实际校验 |
| SH-P2 | 签名失败或版本不匹配时写入 `integrity_events` 并熔断/拒绝服务 | 高 | ☐ | 与 SH-P1 配套 |
| SH-P3 | 规则包版本清单与来源可追溯（如 manifest + 公钥登记） | 中 | ☐ | 见 `spec/HDGP_POLICY_BUNDLE_SIGNING.md` |

### 2.3 防挟持与自审计

| 编号 | 描述 | 优先级 | 状态 | 备注 |
|------|------|--------|------|------|
| SH-H1 | 只读内核：确保核心规则与伦理基线不被运行时覆盖（已通过“无动态注入”实现，可加文档/测试固化） | 高 | ☑ | 当前实现满足，建议增加回归测试 |
| SH-H2 | 自报警机制：当检测到版本/签名异常时，可触发告警或只读降级（伦理基线 §7.3） | 中 | ☐ | 与 SH-P2 联动 |
| SH-H3 | 审计日志防篡改：落盘后只追加、或对接不可变存储 | 中 | ☐ | 与 TD-A1 一起考虑 |

### 2.4 依赖与供应链

| 编号 | 描述 | 优先级 | 状态 | 备注 |
|------|------|--------|------|------|
| SH-D1 | 依赖版本锁定与定期更新（go.sum、依赖审计） | 中 | ☐ | 当前 Go 模块较少，建议固化并定期审查 |
| SH-D2 | 构建可复现性：构建脚本与环境文档化，便于第三方复现二进制 | 低 | ☐ | Beta 对外发布时建议 |

## 三、总览勾选表（按阶段）

- **MVP（当前）**：核心功能与合规测试已就绪；无认证/限流/签名校验，仅适合内网或 PoC。

## 四、使用与更新

- 本文档与 `spec/HDGP_KERNEL_CHECKLIST.md` 互补：Kernel Checklist 偏“内核完整性、文档与治理”，本清单偏“实现层面的技术债与安全”。
- 每完成一项，将对应行的「状态」改为 ☑，并在「备注」中可简要记录完成方式或 PR。
- 新增项可追加到对应小节，并同步更新「三、总览勾选表」。
